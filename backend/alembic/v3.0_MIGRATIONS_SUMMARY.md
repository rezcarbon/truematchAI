## Key Design Decisions

### 1. Safety First
- Backup tables created before ALTER operations
- Reversible downgrade functions
- No data loss operations

### 2. Performance Optimized
- Conditional indices with WHERE clauses
- Composite indices for common queries
- Partial indices to reduce size
- Query-specific index design

### 3. AsyncAlchemy Patterns
- Compatible with psycopg3 async driver
- Uses PostgreSQL-specific features (JSONB, PARTIAL indices)
- Follows project's existing Alembic patterns

### 4. Comprehensive Tracking
- Timeline fields for SLA monitoring
- Audit trails with reviewer tracking
- Consistency metrics for comparisons
- Engagement analytics support

## Dependencies

### System Requirements
- PostgreSQL 13+

- Existing tables: users, resumes, positions, applications

## Migration Notes

### No Data Migration Required
All new columns have appropriate defaults or allow NULL. No existing data requires transformation.

### Foreign Keys
- All new foreign keys use CASCADE/SET NULL semantics
- Existing relationships preserved
- New self-referencing FK on assessments (previous_assessment_id)

### Defaults
- Booleans default to FALSE/TRUE as appropriate
- Enums default to reasonable values (e.g., 'medium', 'daily')
- Numeric defaults to 0
- DateTime fields allow NULL

## Support & Troubleshooting

### Common Issues

**Migration hangs:**
- See "Troubleshooting" section in MIGRATION_v3.0_GUIDE.md
- Check pg_stat_activity for blocking transactions

**Index creation fails:**
- Check existing indices with validation script
- Verify disk space available
- Monitor PostgreSQL logs

**Foreign key violations:**
- Use validation script to identify orphaned records
- See recovery procedures in guide

### Getting Help

1. Review MIGRATION_v3.0_GUIDE.md
