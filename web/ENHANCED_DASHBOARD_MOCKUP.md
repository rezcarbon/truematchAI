## Section Breakdown

### Section 1: Assessment Summary Card

**Visual Structure**:
```
┌──────────────────────────────────────────────────────────────────┐
│ ◀─ [Title] [Position]                     [Match Type Badge]     │
│    [Subtitle] Assessment date                                    │
│                                                                  │
│    [Three-Signal Progression]                                   │
│    ATS (X) → SEM (X) → CAP (X)  Delta: +/-X  [Match Label]    │
│                                                                  │
│    "[Narrative text describing the assessment result and        │
│     key insights about the candidate's fit for the role...]"    │
└──────────────────────────────────────────────────────────────────┘
```

**Color Scheme**:
- Left border: Primary color (blue)
- Background: White/light
- Match badge: Color-coded (amber for hidden gem, green for strong, slate for keyword)
- Text: Default hierarchy (title, subtitle, narrative)

### Section 2: Three-Signal Gauges

**Layout**: 3-column grid (1 col on mobile, 2 on tablet)

**Each Column**:
```
┌──────────────────────┐
│ [Title]              │
│ [Description]        │
│                      │
│    [GAUGE 140px]     │
│                      │
│    [Large Number]    │
│    [Subtitle]        │
└──────────────────────┘
```

**Color Coding**:
- ATS Score: Slate gray
- Delta: Green (positive) or Red (negative)
- Capability: Primary blue

### Section 3: Stats Cards

**Layout**: 3-column grid (1 col on mobile)

**Card Structure**:
```
┌──────────────────────┐
│ [Icon] [Title]       │ 
│                      │
│     [Large Number]   │
│                      │
│ [Subtitle text]      │
└──────────────────────┘
```

**Icons**:
- Assessments: Award (🏆)
- Avg Score: TrendingUp (↗️)
- Active Apps: Zap (⚡)

### Section 4: Delta Visualization

**Chart Type**: Horizontal bar chart (Recharts)

**Features**:
- Two bars: Traditional vs Capability
- Color differentiation
- Axis labels and values
- Tooltip on hover
- Grid background

### Section 5: Strengths & Gaps

**Layout**: 2-column grid (1 col on mobile)

**Left Column - Strengths**:

**Layout**: Vertical list with timeline

2. ✅ Update Profile → `/candidate/profile`
3. 🕐 Preferences → `/candidate/settings`

## Responsive Design Patterns

### Mobile (< 768px)
- All grids: Single column
- Text: Smaller font sizes
- Gauges: Reduced size or removed
- Cards: Full width, stacked vertically

### Tablet (768px - 1024px)
- 2-column grids → 2 columns
- 3-column grids → 2-3 columns (wrap)
- Compact layouts where possible

### Desktop (> 1024px)
- All sections at full width
- Multi-column grids as designed
- Maximum width container: ~1400px

## Dark Mode Considerations

All components use CSS variables defined in theme configuration:
- `@media (prefers-color-scheme: dark)` for OS dark mode detection
- `:root[data-theme="dark"]` for manual dark mode toggle
- Tailwind dark: prefix for component-level adjustments

**Dark Mode Adjustments**:
- Text contrast increased
- Badge backgrounds adapt
- Gauge colors maintained for visibility
- Card borders more visible

## Animation & Interaction

### Hover States
- Cards: `hover:bg-accent/50 transition-colors`
- Buttons: Built-in button hover states
- Links: Underline on hover

### Transitions
- All color changes: `transition-all` (150ms default)
- No heavy animations (keep performance in mind)

### Active States
- Selected/Active badges: Highlighted background
- Buttons: Pressed state styling
